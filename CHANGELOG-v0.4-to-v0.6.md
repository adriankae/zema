# Zema Changelog: v0.4.0 to v0.6.0

Comparison base: `v0.4.0`
Release target: `v0.6.0`

## Summary

This release range takes the browser dashboard from the first complete
dashboard release in `v0.4.0` to a more operationally complete local deployment
and daily-use experience. It adds dashboard-managed Telegram setup, import and
export, richer adherence and onboarding surfaces, network/deployment visibility,
update tooling, and always-available quick status actions for healed and
relapsed locations.

## v0.6.0

### Dashboard Quick Actions

- Added always-available `Healed` and `Relapsed` buttons to the dashboard top
  bar.
- Added a `Healed` picker dialog that lists eligible Phase 1 locations as
  direct action buttons.
- Added a `Relapsed` picker dialog that lists eligible Phase 2+ locations as
  direct action buttons.
- Added clear empty states when no locations are eligible for a quick action.
- Kept quick action buttons visible across Overview, Settings, and all Settings
  subtabs.
- Made the two quick action buttons equal width on desktop and mobile.
- Reused the existing dashboard heal and relapse episode endpoints, preserving
  existing database-side behavior.

### Mobile And Layout Polish

- Adjusted mobile topbar wrapping so quick action controls remain visible and
  tappable on narrow screens.
- Improved spacing for empty list cards such as the Upcoming empty state.

### Tests And QA

- Added dashboard tests for filtered quick action pickers, empty states, Settings
  visibility, and submit behavior for healed and relapsed actions.
- Ran focused dashboard regression coverage: `68 passed`.
- Ran browser QA against a copied database across desktop, mobile, Settings,
  all Settings subtabs, dialog behavior, and end-to-end quick action flows.

## v0.5.0

### Dashboard Telegram Setup

- Added dashboard-managed Telegram bot setup and runtime configuration.
- Added database-backed Telegram bot settings.
- Added Telegram runtime status persistence.
- Added dashboard flows for Telegram token setup, enablement, runtime status,
  and configuration validation.
- Added Telegram runtime support code for backend-managed runtime visibility.
- Added Telegram Docker runtime support through a dedicated Dockerfile and
  start/stop scripts.

### Import And Export

- Added backend import/export support for account-scoped data portability.
- Added CLI backup command support.
- Added dashboard routes and tests for import/export workflows.

### Dashboard Adherence And Onboarding

- Improved dashboard adherence read models and rendering.
- Expanded dashboard onboarding content and first-run guidance.
- Improved adherence visual presentation and supporting tests.
- Added favicon support.
- Improved login/dashboard templates and static dashboard styling.

### Network And Deployment Visibility

- Added a Settings Network tab for deployment/network status.
- Added environment and Compose configuration needed to surface network details.
- Added dashboard tests covering the network settings status tab.

### Update Flow

- Added `scripts/update.sh` for persistent deployment updates.
- Documented update flows for branch/tag based deployment updates.
- Updated `.gitignore` for local runtime artifacts generated by the app and
  tooling.
- Added dashboard asset and route behavior needed for reliable local updates.
