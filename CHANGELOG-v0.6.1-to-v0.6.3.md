# Zema Changelog: v0.6.1 to v0.6.3

Release target: `v0.6.3`

## v0.6.3

### Changed

- Stabilized Docker Compose under the project name `eczema-tracker`.
- Updated `scripts/update.sh` so normal updates keep using the same Compose project name unless explicitly overridden.
- Documented that the Compose project name is part of the production data-persistence contract.
- Updated the dashboard version tag to `v0.6.3`.

### Fixed

- Prevented update flows from accidentally starting against a new empty Postgres volume after a repository directory rename.
- Preserved existing database content, including changed account passwords and Telegram bot settings, by keeping Compose volume names stable across updates.

### Verified

- Added tests that confirm startup bootstrap does not overwrite existing account passwords.
- Added tests that confirm startup bootstrap does not overwrite existing Telegram bot settings.
- Added tests that confirm Compose/update defaults keep the stable project name.

## v0.6.2

### Added

- Added README documentation for the production update flow.
- Documented the recommended `scripts/update.sh` path for pull/build/restart/health-check updates.

### Notes

- `v0.6.2` was a documentation-focused release.

## v0.6.1

### Added

- Added the dashboard version tag under the `Zema` brand in the top bar.
- Added a standalone changelog summary from `v0.4.0` through `v0.6.0`.

### Removed

- Removed the tracked SQLite test database from the repository.
- Removed tracked Python bytecode cache files from the repository.

### Notes

- Local runtime databases and generated bytecode remain ignored by `.gitignore`.
